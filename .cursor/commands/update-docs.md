## Documentation Standards
- Keep files under 100 lines (split into multiple files if needed)
- Use clear, concise language
- Include code examples where relevant
- Ensure all file paths and commands are accurate
- Update any outdated information

## Verification
After updating documentation:
- Review for accuracy and completeness
- Ensure all links and references are correct
- Check that examples still work with current code
